# Canon

Canon is the canonical standards layer for agentic software work. It packages
agent operating rules, repo contracts, and verification gates so Codex and other
agents can follow the same definition of done across repositories.

## Commands

```bash
canon --help
canon doctor
canon check
canon verify
canon agentic-context-source
canon agent "describe the work"
```

## Boundaries

Canon owns standards and proof gates. It does not create agents, route personal
work, or replace Felix, Reeves, Eidos, or Converge:

- Felix creates and maintains agent ecosystems.
- Reeves owns Daniel's chief-of-staff operating picture.
- Eidos owns tracked evidence loops.
- Converge owns target tables, drift monitors, and repair scoreboards.

Canon should stay thin where another source-owned tool already owns behavior.

## Definition of Done

Agent work is not done because the model says it is done. Canon expects:

- relevant repo instructions were read
- dirty worktree state was respected
- changed behavior has deterministic verification
- skipped checks are named with the reason
- final claims cite actual command, test, UI, or service evidence

## Agentic Intelligence

This agent follows the Felix standards: thinking, tools, memory, coordination, and goal orientation.
Memory is substrate, tools are evidence, and work is framed as `have / want / don't want`.
