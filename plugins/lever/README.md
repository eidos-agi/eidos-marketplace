# Lever

Lever is an Eidos Marketplace plugin that routes leverage analysis, durable
learning, and agent-build decisions to the live Lever CLI. It is the agent that
asks whether the work creates power, not just motion.

Lever should be used when the work is not merely a task, but a chance to create
more durable agency: reusable systems, specialist agents, better decisions,
compounding relationships, infrastructure, or optionality.

Lever's posture is sharp: find the constraint, build the reusable thing, kill
weak loops, upgrade proven loops, and record the durable lesson.

Use Lever for three hard calls:

- whether a move is leverage or dressed-up motion
- whether repeated work deserves a bounded specialist agent
- whether a correction should become durable behavior

## Commands

```bash
lever doctor
lever manifesto
lever score "turn repeated triage into a bounded agent" --leverage-type automation --constraint "manual routing consumes founder attention" --compounding 5 --reversibility 4 --control 4 --optionality 4
lever agent-build "recurring inbox triage" --role "Inbox leverage scout" --repeatability 5 --boundedness 4 --proof-loop 5 --memory-fit 4 --refusal-boundary 4 --approval-risk medium
lever review --focus "current bottleneck"
lever agent "describe the work" --leverage-type optionality
lever learn "durable correction" --source "human correction" --evidence "short evidence" --leverage-type decision --confidence high
lever lessons --limit 5
lever distill --limit 20
lever improve --focus "Lever itself" --limit 20
```

If `lever` is not on `PATH`, this local installation can use the source checkout
at `/Users/dshanklin/repos-personal/lever/.venv/bin/lever`.

## Boundary

This plugin is a thin routing layer. Durable behavior, learning state, tests,
and source-owned CLI logic belong in the canonical Lever source repo.
